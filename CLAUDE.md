# CLAUDE.md — Apply Co-Pilot

Context for Claude Code sessions in this repo. Read this first; don't re-derive it.

## What this is
A **single-file, 100% client-side** web app: upload a résumé once → paste a job description → get an **honest ATS fit score** (original vs. projected-after-tailoring), then a **custom, ATS-optimized résumé + cover letter** as a real **vector PDF** with clickable links and selectable text. No backend, no build, no AI/LLM, no API keys. The only runtime network use is four CDNs: **pdf.js** (résumé import), **pdfmake + its bundled fonts** (PDF export), **three.js** (the 3D hero sphere), and **Google Fonts** (type). Everything else runs in the browser; data lives in `localStorage`. Repo is public.

## Architecture / data flow
`upload → M.raw (localStorage) → parseResume() → analyze(company,role,jd) → CUR{orig,tail,buckets,need,have,cv} → render* → resumeDocDef()/coverDocDef() → pdfMake.createPdf().download()`

The flow is **Fit-first**: `run()` plays a staged progress animation then `tab("kw")` smooth-scrolls to the **① ATS Score** section showing the honest **original** score before the tailored résumé. The UI is a **vertical-scroll single page** (not tabs); `tab(t)` scrolls to `#pane-<t>` (via `window.__lenis.scrollTo` when Lenis loaded, else native smooth scroll). Section/nav order (labels → `data-t`): hero (`top`) · ⓪ Master Résumé (`mr`) · ◆ Job Description (`job`) · ① ATS Score (`kw`) · ② Custom Résumé (`cv`) · ③ Cover Letter (`cl`) · ④ LinkedIn Outreach (`out`) · ⑤ Job Tracker (`trk`) · ⑥ Find Jobs (`find`, last). All render targets keep their original element IDs.
- **Tips → accept→apply (offline, no AI):** `detRecs()` returns structured `{t, add[]}`; the two keyword tips render with a `＋ Add`/`✓ Added` toggle (`acceptTip(i)` → `CUR.added`). `updateProjected()` re-scores live treating accepted keywords as `have`. **`viewTailored()`** (the "→ View tailored résumé" button) sets `CUR.cv = buildDisplayCv()` (clones `CUR.cvBase`, appends an "Additional Skills" group with accepted keywords) then renders — so the CV reflects accepted edits only when viewed. The AI rewrite of bullets remains the deferred backend.
- **1-page:** `finalizeOnePage(cv)` (called in `resumeDocDef`) trims the lowest-JD-relevance bullets (already sorted) until `cvWeight ≤ 4400`; `analyze` also caps exp ≤6 bullets/role, projects ≤5×3. Experience renders resumeworded-style: **role bold + dates right (line 1), `org · location` italic (line 2)**.

Key functions in `apply_copilot_app.html`:
- **Parser**: `parseResume(raw)` → `{name,email,phone,location,linkedin,github,summary,skills[],experience[],projects[],education[],certifications[]}`. `parseProjects` → `{name,tech,link,bullets[]}` (splits real bullets on `– — • ·`; detects a `github.com/...` URL, or `"__profile__"` sentinel when the résumé only says the word "GitHub" → links to the profile). `parseEdu` → `{institution,degree,dates,location}` (or `{raw}` fallback) for a two-column render. `parseCerts` → `string[]`. Other helpers: `parseSkills`, `parseExp`, `splitBullets`, `splitCsv`, `cleanName`, `normRange`/`normDate` (ATS date normalization), `tidy()`/`deLig()` (normalize dashes/quotes + strip ligatures & icon-font garbage). **Parser is line-aware** (uses pdf.js `hasEOL` newlines, kept by `parseResume`'s `t` normalization): skills (labels with/without colons, tab-separated, bullet-prefixed; wrapped lines append to prev group), experience (`splitHeader` role/org/location; `•`-with-date = entry vs `◦` = sub-bullet; multi-line headers), education (`parseEduEntry`, gpa-aware), projects (`projHeadLine`/`parseProjHead`). **Each falls back to the old flattened single-line regex when a section has <2 newlines** (legacy masters / pasted text). Section headers detected **anchored to line starts** so "experience" inside a summary sentence doesn't false-trigger. **Wrapped bullets MERGE** (a markerless continuation line is appended to the previous bullet, not pushed as a new one — this both prevents fragmentation AND keeps the PDF to ~1 page). `parseResume` runs a **`denoise()`** on every section: strips `page X of Y` + the candidate's own name (running headers from re-uploaded multi-page generated PDFs). **`location` comes from the CONTACT block only** (never a job's city — empties for résumés with no contact location rather than guessing "Mysore"). Adds an **`achievements`** field (Awards/Honours/Coding Profile → rendered as its own section). Bullets capped (exp ≤6/role, projects ≤5×3) in `analyze` for 1-page. **Stress-tested via a 10-case node harness** incl. the critical *re-upload-a-generated-2-page-PDF* case — keep that harness shape when changing the parser; test WRAPPED bullets, not just one-line-per-bullet fixtures.
- **Fit / scoring (deterministic, defensible)**: `bucketSkills(jd, src)` splits BANK matches into `{reqHave,reqMiss,prefHave,prefMiss}` via required/preferred cue phrases. `reqYears(jd)`, `rankOf(text)` (0–8 seniority ladder), `candidateYears(exp)` (**interval-merges** date ranges so concurrent roles don't double-count). `scoreArrangement(buckets,need,have,structure,zone)` = weighted (kw .50 / seniority .30 / structure .10 / keyword-placement .10) with **multiplicative knockouts** (years < ½ required → ×0.40; seniority gap ≥3 → ×0.45; <34% required → ×0.6). Honest floor 2, cap 99 — a 1-yr résumé vs a VP role scores ~7–15, not a floored 45. `detRecs()` for tips.
- **Tailoring**: `analyze()` returns both `orig` (master arrangement) and `tail` (tailored: bullets/skills reordered toward JD via `bulletScore`/`jdHas`, `tailoredSummary()` 3–4 sentences). The delta comes only from surfacing matched keywords into the summary/skills zone (honest, bounded). `detCover()` builds the cover letter.
- **Render**: `resumeDocHTML(cv)` (real `<a>` links; bullet-point projects with GitHub link; two-column education via `.role`/`.sub2`; certs as `<ul>`). `renderKW()` shows two score cards + breakdown bars + knockout banners + req/pref chips. `toggleEdit(id,btn)` flips the `.doc` between read-only (links clickable) and `contenteditable` (for fixing text).
- **PDF (real vector, no print dialog)**: `resumeDocDef(cv)` / `coverDocDef(cv,cover)` build **pdfmake** docDefinitions; `makePdf(which)` → `pdfMake.createPdf(def).download(name+'.pdf')`. Clickable links via `{text,link}`, selectable text, native `ul` bullets, role-left/date-right `columns` (single logical line = ATS-safe), section rules via `canvas` line. `baseFont(cvWeight(cv))` adaptively shrinks font 10.5→8pt to keep ~1 page. **No `window.print()`** (that's what added the URL/timestamp watermark) — `printFallback()` only runs if pdfmake fails to load. Do NOT use pdfmake `.html()` or html2canvas (rasterizes → ATS can't read it).
- **State**: `localStorage` keys `apply_master_v3` (`M={raw,name}`) and `apply_copilot_log_v3` (`LOG=[]`).

## Hard product constraints (decided — don't undo without asking)
- **No AI / no LLM / no keys / no Overleaf / no LaTeX.** Pure deterministic. (We tried WebLLM + LaTeX; both removed.) Scoring/tailoring is rule-based vanilla JS — no network analysis, no scraping (a pure client app can't fetch arbitrary job URLs anyway: CORS + bot-blocks. Decided 2026-06: stay offline; make the *analysis of pasted JD text* smarter instead).
- **The custom résumé + cover letter are the USP.** ATS-optimized, ~1 page, real hyperlinks, never invents facts (only reorders/rephrases real content).
- **The ATS score must be legitimate, not inflated.** Show the honest original score first; knockouts (years/seniority/core-skills) genuinely crush unrealistic applications. Never re-add a score floor.
- **Master = just upload** (PDF/TXT). No structured-field forms, no "auto-fill" step.
- **PDF = real vector via pdfmake** (programmatic, clickable links + selectable text, direct `.download()` so there's **no browser print header/timestamp watermark**). *(Reversed 2026-06 the old "native print only" rule — native print rasterized on some drivers and stamped a watermark.)* Output must stay single-column with linear text order; never rasterize (no html2canvas/`.html()`).
- **UI = futuristic vertical-scroll, dark.** Single scrolling page (NOT tabs): a fixed **three.js** fresnel-glow icosahedron + starfield hero behind the content (`#hero3d`, reacts to scroll; CSS-gradient `#heroFallback` shown + sphere skipped on reduced-motion/mobile/weak-GPU), a sticky `.nav` whose links smooth-scroll to `#pane-*` sections (`tab(t)`), a scroll-progress bar, and IntersectionObserver `.reveal` fade-ins. IBM Plex Mono + Space Grotesk, glass `.panel`s with corner brackets, status chips; honors `prefers-reduced-motion`. The résumé `.doc` stays **light/white** and is `z-index:2` above the `pointer-events:none` canvas (it's "paper" and must print clean). *(three.js pinned to UMD global `0.158.0` — r160+ removed the global build, and the ES-module/import-map path does NOT load from `file://`, which would break local `Invoke-Item` testing.)*
- Privacy: nothing leaves the browser. Keep it that way.

## Conventions
- Match the existing terse, single-line-function vanilla-JS style. No frameworks. Runtime CDNs: pdf.js; pdfmake (+`vfs_fonts.js`, pinned `0.2.12` on jsDelivr — load order matters: `pdfmake.min.js` then `vfs_fonts.js`); **three.js UMD global `0.158.0`** (`build/three.min.js` — NOT a module/import-map, so it works from `file://`); **Lenis `1.3.23`** (UMD global, smooth-scroll; `tab()` uses `window.__lenis.scrollTo`; skipped on reduced-motion); Google Fonts. New CDN libs only if essential. ⚠️ `deLig()` (ligature + Unicode-PUA/zero-width icon-font stripping) MUST use `\uXXXX` escapes, never literal PUA/zero-width chars — a full-file Write corrupts the literal PUA range into one that also strips ASCII hyphens (mangles "cross-functional" etc.).
- Code/UI text often uses `é` (résumé) and emoji — fine; files are UTF-8.
- Keep it one file; don't split into modules.
